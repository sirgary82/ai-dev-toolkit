The Model Context Protocol lets Claude Code connect to external tools and data sources as if they were native capabilities. Instead of copying output from a browser, terminal, or database client and pasting it into Claude, MCP servers let Claude query those sources directly during a session.

These are the servers that consistently provide the most value for day-to-day development work.

---

## How MCP servers are configured

All MCP servers are added to `~/.claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "server-name": {
      "command": "npx",
      "args": ["-y", "package-name"],
      "env": {
        "SOME_TOKEN": "your-token"
      }
    }
  }
}
```

Restart Claude Code after adding a new server.

## GitHub MCP

**What it does:** Lets Claude Code query GitHub directly — read issues, PRs, repository content, CI status, comments, and more. Works alongside the `gh` CLI but gives Claude a richer, structured interface to GitHub's API.

**Setup:**

```json
"github": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-github"],
  "env": {
    "GITHUB_PERSONAL_ACCESS_TOKEN": "your-github-pat"
  }
}
```

Create a token at GitHub → Settings → Developer settings → Personal access tokens. Scope: `repo`, `read:org`.

**Best uses:**
- "What are the open issues labelled `bug` assigned to me?"
- "Summarise the comments on PR #247 — what's the main disagreement?"
- "What did the CI failure on this branch say?"
- "Find all PRs that touch OrderService and are waiting for review"

## NuGet MCP (Official, Microsoft)

**What it does:** Gives Claude direct access to NuGet registry data — current package versions, dependency resolution, and vulnerability information. Solves the LLM knowledge-cutoff problem for .NET packages: instead of suggesting the version from training data, Claude queries the live registry.

**Setup (VS Code):**

```json
// .vscode/mcp.json
{
  "servers": {
    "nuget": {
      "type": "stdio",
      "command": "dnx",
      "args": ["nuget-mcp"]
    }
  }
}
```

Requires .NET 10 SDK. In Visual Studio 2026, it's built-in under the Copilot Chat Tools menu.

**Best uses:**
- "Add the latest stable version of MediatR to this project" — resolves against live registry, not training data
- "Update all packages to the latest compatible versions"
- "Which of our packages have known vulnerabilities and what version fixes them?"

See [LLM Version Staleness](/context-management/llm-version-context/) for the full picture on why this matters and what to do without MCP.

## package-registry-mcp (npm + NuGet + PyPI + Cargo)

**What it does:** A community MCP server covering five package registries. Useful for teams working across ecosystems, or for fullstack repos with both .NET and npm dependencies.

```bash
claude mcp add -s user package-registry npx package-registry-mcp
```

**Best uses:**
- "What is the latest stable version of react-hook-form?"
- "What versions of FluentValidation are available for .NET 9?"
- "Are there any security advisories for these packages?" (exposes GitHub Security Advisories)

## Filesystem MCP

**What it does:** Gives Claude structured access to your local filesystem beyond the current working directory. Useful when you need Claude to read configuration files, logs, or documentation that lives outside the project directory.

**Setup:**

```json
"filesystem": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-filesystem",
    "/Users/you/Documents",
    "/Users/you/projects"
  ]
}
```

List the directories you want Claude to be able to read.

**Best uses:**
- Reading log files from a running service without copying them
- Accessing shared documentation that lives outside the repo
- Reading configuration from multiple projects in a single session

<Callout type="note">
  Only add directories you're comfortable with Claude reading. Avoid adding directories that contain sensitive files (password managers, SSH keys, `.env` files).
</Callout>

## PostgreSQL / SQL Server MCP

**What it does:** Lets Claude query your database directly — read schema, run SELECT queries, and reason about data without you copying table structures or query results.

**PostgreSQL:**

```json
"postgres": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-postgres",
    "postgresql://user:password@localhost:5432/your_db"
  ]
}
```

**Best uses:**
- "What columns are in the orders table and what are their types?"
- "Write a query to find all customers who haven't placed an order in 90 days"
- "Why is this query slow? Here's the query — can you analyse it against the schema?"
- "Generate an EF Core entity class for the products table"

<Callout type="warning">
  Connect to a development or staging database, not production. An MCP-connected Claude session can read data — ensure it connects only to environments appropriate for that.
</Callout>

## Brave Search MCP

**What it does:** Gives Claude the ability to search the web in real time during a session — finding documentation, error explanations, library examples, or Stack Overflow answers without you leaving the terminal.

**Setup:**

Get a free API key at [brave.com/search/api](https://brave.com/search/api).

```json
"brave-search": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-brave-search"],
  "env": {
    "BRAVE_API_KEY": "your-key"
  }
}
```

**Best uses:**
- "Search for the EF Core 8 migration guide and summarise the breaking changes"
- "Find the current syntax for this NuGet package — my code is using the v3 API"
- "Look up this error message and tell me the most common cause"
- Researching a library before adding it as a dependency

This is the MCP that has the highest day-to-day frequency of use. Not having to leave the terminal to search for documentation keeps the session focused.

## Slack MCP

**What it does:** Lets Claude read Slack channels and threads — useful for understanding the context of a decision, finding a discussion about a bug, or summarising what was agreed in a thread.

**Setup:**

```json
"slack": {
  "command": "npx",
  "args": ["-y", "@modelcontextprotocol/server-slack"],
  "env": {
    "SLACK_BOT_TOKEN": "xoxb-your-token",
    "SLACK_TEAM_ID": "your-team-id"
  }
}
```

Requires creating a Slack app in your workspace with `channels:history` and `channels:read` scopes.

**Best uses:**
- "Find the Slack discussion where we decided to use Redis for session storage"
- "Summarise what the team agreed about the API versioning approach in #backend this week"
- "What was the context for the decision to deprecate the old auth endpoint?"

This MCP turns Slack threads into searchable context rather than things you have to dig through manually.

## Building your own MCP server

For organisation-specific context — your internal API docs, your custom tooling, your architecture documentation — you can build a simple MCP server. Claude Code will then be able to query it the same way it queries the standard servers.

See [MCP for Repo Context](/context-management/mcp-repo-context/) for a full example of building a docs MCP server in TypeScript.

The pattern for any internal tool:

```typescript
import { Server } from '@modelcontextprotocol/sdk/server/index.js';

const server = new Server({ name: 'internal-tools', version: '1.0.0' });

server.setRequestHandler(ListToolsRequestSchema, async () => ({
  tools: [{
    name: 'get_service_docs',
    description: 'Get the documentation for an internal service',
    inputSchema: {
      type: 'object',
      properties: {
        service: { type: 'string', description: 'Service name' }
      }
    }
  }]
}));

server.setRequestHandler(CallToolRequestSchema, async (request) => {
  if (request.params.name === 'get_service_docs') {
    const docs = await fetchDocsForService(request.params.arguments.service);
    return { content: [{ type: 'text', text: docs }] };
  }
});
```

## MCP server discovery

The MCP ecosystem is growing rapidly. To find servers beyond the ones listed here:

- **[github.com/modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers)** — the official reference server collection
- **[mcp.so](https://mcp.so)** — community directory of available MCP servers
- Search npm for `@modelcontextprotocol/server-*` for official packages

When evaluating a community MCP server, check: who maintains it, what permissions it requests, and whether it has been reviewed. Only install servers from sources you trust.
