## Frontend-specific tips

- **Open related components.** Copilot reads open tabs. When building a new component, open a similar existing component alongside it.
- **Describe the component contract first.** Write the prop types and a JSDoc comment, then let Copilot fill in the implementation.
- **Ask in terms of behavior.** "Add an onClick handler that..." is clearer to Copilot than "make this button do something."
- **Generate test skeletons.** Copilot is excellent at generating `describe`/`it` blocks when you already have the component open.
- **For Claude with React monorepos:** Add an AGENTS.md in each package directory. Include the component library and state management library in use — Copilot and Claude will default to React patterns, not your specific stack.
- **Paste the design system tokens.** When asking Claude to build a component, including the relevant CSS variables or Tailwind config gives it the actual values to use, not generic placeholders.
