## Backend-specific tips

- **Open the relevant files.** Copilot reads open tabs. When generating a new service, open the interface, the data model, and an existing similar service.
- **Use XML doc comments as prompts.** Write the `/// <summary>` first and let Copilot generate the method body.
- **Scope to the class, not the project.** Ask "add a method to this class that..." rather than "build me a feature that...".
- **For Claude Code on .NET solutions:** `cd` into the specific project directory before starting a session. Scoping Claude Code to one project produces better results than letting it explore the full solution.
- **Write one AGENTS.md per .NET project** (not per solution). Each project has its own patterns, dependencies, and constraints.
